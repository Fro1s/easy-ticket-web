# This is NOT the Next.js you know

This version has breaking changes — APIs, conventions, and file structure may all differ from your training data. Read the relevant guide in `node_modules/next/dist/docs/` before writing any code. Heed deprecation notices.

## Commands

```bash
pnpm dev          # Start development server (localhost:3000)
pnpm build        # Build for production
pnpm lint         # Run ESLint
pnpm test         # Run Vitest tests
pnpm test -- path/to/file.test.ts   # Run a single test file
pnpm generate:api # Regenerate API client from OpenAPI spec (requires backend running at localhost:3001)
```

## Architecture

### Route Groups
Routes are organized into layout groups under `src/app/`:
- `(app)/` — authenticated user routes (my tickets, my account)
- `(shop)/` — public e-commerce flow (event listing, event detail, checkout, success)
- `(marketing)/` — public marketing pages
- `admin/` — admin dashboard
- `painel-produtor/` — producer dashboard
- `auth/`, `entrar/`, `cadastro/` — authentication pages

### API Client
The backend API client in `src/generated/api.ts` is **auto-generated by Orval** from the OpenAPI spec at `http://localhost:3001/docs/json`. Never edit this file manually — run `pnpm generate:api` to update it. It exports TanStack Query hooks (`useQuery`/`useMutation`) for every endpoint.

The custom axios instance (mutator) is at `src/lib/api.ts`. Auth headers, base URL, and interceptors live there.

### Data Fetching
All server data fetching uses the generated hooks from `src/generated/api.ts`. TanStack Query is configured in `src/lib/query-client.ts`.

### UI Components
- Base components (`src/components/ui/`) are from **shadcn/ui** — copy-pasted, not imported from a package.
- Business components live in `src/components/`.
- Styling uses Tailwind CSS with `cn()` from `src/lib/utils.ts` (clsx + tailwind-merge).

### Key Libraries
- **React 19** and **Next.js** (check version in package.json — not stable Next.js 13/14)
- **TanStack Query v5** for data fetching
- **Framer Motion** for animations
- **sonner** for toasts
- **react-imask** for input masking
- **date-fns** for date formatting
- **jspdf** + **qrcode** for ticket PDF/QR generation
- **@zxing/library** for QR code scanning
